# ControlForge Structured Text v1.4.0

**Release Date:** March 15, 2026

## Overview

Version 1.4.0 adds XML/TwinCAT/CoDeSys file highlighting, five new diagnostic categories, signature help for function/FB calls, cross-file rename, and a test suite that grew to 541 tests.

## What's Added

- **XML export highlighting**: ST syntax injected into `<Declaration>`, `<ST>`, `<Implementation>`, and `<Interface>` elements in TwinCAT/CoDeSys export files (plain content and CDATA-wrapped)
- **Hardware address literals** (`%IX0.0`, `%QW1`, `%MD100`, etc.) highlighted as `constant.other.hardware-address.st`
- **Pragma/attribute blocks** (`{attribute 'xxx'}`, `{IF}`, `{ENDIF}`) highlighted as `meta.pragma.st`
- **SFC action qualifier keywords** (`N`, `S`, `R`, `L`, `D`, `P`, `SD`, `DS`, `SL`) highlighted as `keyword.other.sfc-qualifier.st`
- **OOP keywords**: `CLASS`, `END_CLASS`, `INTERFACE`, `END_INTERFACE`, `METHOD`, `END_METHOD`, `PRIVATE`, `PROTECTED`, `PUBLIC`, `INTERNAL`, `END_NAMESPACE`, `PROPERTY`
- **`ANY_CHAR` / `ANY_CHARS`** generic types added to `storage.type.st`
- **Named parameter scoping**: FB/function call parameters (`Param :=`, `Param =>`) scoped as `variable.parameter.st`
- **Signature help**: triggers on `(` and `,`; highlights active parameter; covers standard and workspace-defined functions and FBs
- **Constant assignment diagnostic**: error when assigning to a `CONSTANT`-qualified variable
- **Array bounds diagnostic**: error for out-of-bounds access with constant/literal indices (1-D, multi-dimensional, zero-based, negative-lower-bound)
- **FOR loop bounds validation**: `BY 0` → error; reverse range → warning; start equals end → hint
- **`:=` vs `=` confusion diagnostic**: bare `=` in statement position flagged with "Replace '=' with ':='" quick fix; `:=` in condition flagged with "Replace ':=' with '='"
- **Cross-file rename**: F2 now renames symbols across all workspace files

## What's Changed

- Bumped `@types/node` to `^25.0.0`, `glob` to `^13.0.0`; added `skipLibCheck` for TS 5.x compatibility
- Standardized manual test fixture naming (removed `test_`/`test-` prefixes)
- Removed `docs/QA_CHECKLIST.md` and `images/README.md`

## What's Fixed

- Multi-line FB calls with `)` on its own line falsely flagged as "Unmatched closing parenthesis"
- Output named parameters (`Param =>`) falsely flagged as undefined identifiers
